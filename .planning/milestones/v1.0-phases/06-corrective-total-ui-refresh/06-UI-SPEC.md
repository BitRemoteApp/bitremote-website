## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Hairline internal spacing, icon-text separation |
| sm | 8px | Compact label and badge spacing |
| md | 16px | Default control padding and card interior rhythm |
| lg | 24px | Standard block gaps inside sections |
| xl | 32px | Card groups, hero content stacks |
| 2xl | 48px | Major subsection separation |
| 3xl | 64px | Section-to-section spacing on desktop |

Exceptions: minimum interactive target height may use 44px; primary navigation height should resolve to 48px minimum.

Section rhythm contract:
- Use `48px` vertical gaps on mobile and `64px` on desktop between major homepage sections.
- Do not separate every section with a divider. Use whitespace first; dividers are optional emphasis, not a default scaffold.
- Keep content width calm and readable: primary text blocks should stay within `60–68ch`; dense capability grids can expand visually but not text measure.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.3 |
| Heading | 20px | 600 | 1.2 |
| Display | 40px | 600 | 1.08 |

Typography contract:
- Use only these four sizes for Phase 6 UI surfaces. Do not reintroduce uppercase monospace-style micro-label treatment.
- Display text may scale fluidly, but its desktop target is `40px` and it must remain visually controlled rather than oversized.
- Body copy should read as calm product prose, not ad copy. Paragraphs should typically stay within `2–4` lines on desktop.
- Wordmark styling must be clean and static. No glitch, pixelation, faux-terminal tracking, or novelty motion.

**Source:** `.impeccable.md` plus existing `--font-sans` token in `src/app/globals.css`

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `#0a0a0a` | Page background, large surfaces, canvas |
| Secondary (30%) | `#f8f8f8` / `#141414` | Cards, nav bar, demo frame, grouped panels |
| Accent (10%) | `#2563eb` / `#3b82f6` | Primary CTA, selected nav state, focused controls, key product highlights |
| Destructive | `#dc2626` / `#f87171` | Destructive actions only; not expected in primary marketing flow |

Accent reserved for: primary App Store CTA, active navigation item, active locale selection, active demo controls, key proof points inside the product demo, inline text links, and focus-visible outlines.

Color contract:
- Neutrals do most of the work. Blue is for commitment, state, and priority, not decoration.
- Remove accent-on-hover inversion patterns that make controls feel playful or terminal-like.
- Glass effects may remain only where they support depth quietly. Avoid generic glossy SaaS-card treatment.
- Do not introduce additional brand hues, neon tints, purple bias, or rainbow gradients.

**Source:** `DSGN-03`, `.impeccable.md`, `src/app/globals.css`

## Information Architecture Contract

Phase 6 is a structural correction. The homepage must no longer be organized around a standalone screenshot showcase.

Required homepage order:
1. Sticky navigation with clear active state, language switch, and App Store CTA.
2. Hero with plain brand lockup, value proposition, supporting paragraph, primary CTA, and platform support proof.
3. Interactive product demo section placed immediately after the hero or integrated into the hero composition.
4. Supported Downloaders section near the top of the page.
5. Product capabilities / benefits section.
6. How it works / quickstart section.
7. BitRemote Plus section.
8. FAQ section.
9. Footer and legal/support links unchanged in coverage.

Structural rules:
- `AppShowcaseClient` must not remain as a standalone section in this phase.
- Screenshots may appear only as supporting assets inside the demo or feature storytelling.
- Supported Downloaders must be presented as trust-building product coverage, not a buried appendix.
- The page should feel composed through hierarchy and grouping, not through repeated divider lines and repeated box treatments.
- The first focal point on load is the hero value proposition and primary CTA; the interactive demo is the immediate secondary proof block that validates that claim.

**Source:** `06-CONTEXT.md` decisions D-08 through D-15, `.impeccable.md`, `ROADMAP.md`

## Component Contract

### Navigation
- Replace `TextTabsNav` with a calmer app-like navigation bar.
- Use persistent active-state styling for the current section/page. The current “hover only, never active” behavior is rejected.
- Locale switching should use a rounded popover or list panel with a visible selected state. No bracket wrappers.
- Mobile navigation should remain simple and compact, not terminal-like.

### Primary CTA
- Replace `TextButton` with a filled or softly tinted button treatment.
- Primary CTA shape: rounded rectangle using existing radius tokens.
- Hover behavior: subtle surface lift, tint shift, or shadow change only. No bracket reveals, no abrupt full inversion.
- Secondary CTA, if present, should be quiet text or outline treatment rather than another novelty button style.

### Brand Lockup
- Replace `BitRemoteWordmark` with a static lockup or clean typographic mark.
- No animated filters, no pixel artifacts, no glitch loops, no motion-dependent rendering split.

### Product Demo
- Introduce a modern interactive demo or simulator that communicates real product behavior.
- Demo styling must feel like a sober native app preview: layered surfaces, clear states, restrained motion, readable data density.
- Use interaction to explain product value, not to imitate a terminal.
- Empty and fallback states must look intentional; the demo cannot collapse into a blank shell.

### Supported Downloaders
- Add a dedicated homepage section using the existing downloader domain data.
- Present each downloader as a clean capability tile, list row, or segmented matrix with links to landing pages where appropriate.
- Prioritize scan speed and trust over decorative illustration.

### Existing Section Restyling
- Bento cards, FAQ rows, badges, and quickstart steps should be normalized toward the same calmer surface language.
- Uppercase-heavy labels should be reduced. Use sentence case for most user-facing UI unless legal or platform naming requires otherwise.
- Dividers should be reduced or removed where spacing alone creates enough structure.

## Interaction Contract

Motion and behavior rules:
- Keep existing reduced-motion compliance and transform/opacity-only animation rules from Phases 2 and 5.
- Use entrance motion sparingly: section reveals and demo state transitions only.
- Hover states should communicate precision, not theatrics. Prefer `150–200ms` transitions.
- Selected states must be visible without hover for nav items, locale choices, and demo controls.
- Focus rings remain accent-colored and must stay visible in both themes.

State rules:
- Demo controls need explicit default, hover, active, focus, empty, and unavailable states.
- Navigation needs default, hover, active, pressed, and mobile-collapsed states.
- Supported downloader links need default, hover, focus, and current-page-safe behavior.

Anti-patterns banned in this phase:
- Brackets as decorative UI framing
- Glitch, pixel, or scanline effects
- Hover-only discoverability
- Decorative motion that does not teach product value
- Large gradient backdrops competing with content hierarchy

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Download on the App Store |
| Empty state heading | No active downloads in this preview |
| Empty state body | BitRemote keeps your download clients within reach from iPhone, iPad, and Mac. Connect a supported downloader to monitor progress, queues, and status in one place. |
| Error state | Preview unavailable right now. Continue below to review supported downloaders and key BitRemote capabilities. |
| Destructive confirmation | none — no destructive actions in Phase 6 marketing UI |

Copy contract:
- Use direct product language. Avoid playful hacker phrasing, faux-terminal metaphors, or self-conscious cleverness.
- Section headings should explain capability or trust, not act as abstract taglines.
- Downloader section copy should emphasize compatibility and confidence.

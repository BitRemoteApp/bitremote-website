## Spacing Scale

Source: Phase 1 globals.css + current page.tsx patterns. Standard 8-point scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, bracket decorators, inline padding |
| sm | 8px | Card internal gap, list item spacing |
| md | 16px | Default element spacing, mobile horizontal padding (`px-4`) |
| lg | 24px | Card padding, bento grid gap, section internal gap |
| xl | 32px | Section vertical padding, between feature cards |
| 2xl | 48px | Hero vertical breathing room, section top padding on mobile |
| 3xl | 64px | Section vertical padding on desktop |

**Exceptions:**
- `scroll-padding-top`: 5.25rem (84px) — established Phase 1 globals.css; do not change
- Nav bar height: ~2.8rem inner height + border (sticky top-0 z-10) — preserve as-is
- Touch target minimum: 44px height for all interactive elements (App Store CTA button)
- Platform badge strip height: auto-sized to badge image at 1x (logical pixel dimensions match Apple badge guidelines ~40px tall)

## Typography

Source: Phase 1 globals.css. Body line-height: 1.55 (`body { line-height: 1.55 }`). Font: `var(--font-sans)`.

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 | 1.55 | `text-base` (global default) |
| Label / Nav | 14px | 400 | 1.5 | `text-sm uppercase tracking-wide` |
| Section heading | 20px | 600 | 1.2 | `text-xl font-semibold` |
| Display / Hero headline | clamp(32px, 4vw, 48px) | 600 | 1.05 | `text-[clamp(2rem,4vw,3rem)] leading-[1.05] tracking-[-0.02em]` |

**Phase 4 typography usage by section:**

- **Hero headline**: clamp display size, weight 600, line-height 1.05, tracking −0.02em — mirrors existing `page.tsx` tagline pattern
- **Hero subheadline**: 16px body, weight 400, `text-text-secondary`, max-width 62ch
- **Section heading labels** (Features, Quickstart, Pricing, FAQ): 16px, weight 400, `uppercase tracking-wide text-accent` — established pattern in current `page.tsx`
- **Feature card title**: 20px (text-xl), weight 600, line-height 1.2
- **Feature card body**: 16px, weight 400, line-height 1.55, `text-text-secondary`
- **Bento card label / tag** (if used): 14px, weight 400, uppercase, `text-text-muted`
- **Pricing note**: 16px, weight 400, `text-text-secondary`
- **FAQ question**: 14px (text-sm), weight 400, uppercase, `tracking-wide text-text-primary` — existing FaqAccordion pattern
- **Platform badge strip**: no typographic element — image-only badges

## Color

Source: Phase 1 globals.css — full token set. Exact hex values are theme-aware via `prefers-color-scheme`.

| Role | Dark value | Light value | CSS Token | Usage |
|------|-----------|-------------|-----------|-------|
| Dominant (60%) | `#0a0a0a` | `#ffffff` | `var(--color-bg)` | Page background, hero background |
| Secondary (30%) | `#141414` | `#f8f8f8` | `var(--color-surface)` | Feature bento card background |
| Surface-alt | `#1e1e1e` | `#f0f0f0` | `var(--color-surface-alt)` | Card border, hover elevated surface |
| Border | `#2a2a2a` | `#e5e5e5` | `var(--color-border)` | Card stroke, `<hr>` dividers, nav border |
| Text primary | `#f5f5f5` | `#0a0a0a` | `var(--color-text-primary)` | Headings, nav items |
| Text secondary | `#a3a3a3` | `#6b7280` | `var(--color-text-secondary)` | Body copy, card descriptions, subheadlines |
| Text muted | `#6b6b6b` | `#9ca3af` | `var(--color-text-muted)` | Helper text, timestamps |
| Accent (10%) | `#3b82f6` | `#2563eb` | `var(--color-accent)` | See reserved list below |
| Accent hover | `#60a5fa` | `#1d4ed8` | `var(--color-accent-hover)` | Accent hover state only |
| Accent subtle | `#1e3a5f` | `#dbeafe` | `var(--color-accent-subtle)` | Text selection highlight |
| Glass 92% | — | — | `var(--bg-glass-92)` | Sticky nav background (existing) |
| Glass 88% | — | — | `var(--bg-panel-88)` | Feature card glassmorphism background |

**Accent reserved for:**
1. App Store download CTA button (primary TextButton variant)
2. Section heading labels (accent color text on upper-case labels)
3. Focus ring on all interactive elements (`:focus-visible` outline)
4. Navigation link hover and active states (bg-accent + text-bg inversion)
5. Text links within body copy (existing `a { color: var(--color-accent) }` rule)

Accent is NOT used for: decorative dividers, card borders, image frames, background fills, section backgrounds.

**Glassmorphism contract (FEAT-02):**
- Dark theme: `background: var(--bg-panel-88)` + `backdrop-blur-md` + `border border-[var(--color-border)]` + `shadow-card`
- Light theme: same tokens — `--bg-panel-88` resolves to a near-white translucent value via `color-mix`; `--shadow-card` provides soft lift
- No additional color tokens needed; glass effect uses existing `--bg-panel-88` and `--shadow-card`

**`HeroSection` visual contract:**
- Full-width section, `id="top"`, no explicit height constraint — content determines height
- Internal layout: flex column, gap-lg, max-width container (`max-w-6xl mx-auto px-4`)
- Headline: display typography, `text-text-primary`
- Subheadline: body size, `text-text-secondary`, `max-w-[62ch]`
- CTA: `TextButton` primary variant (accent-colored bracket link) linking to App Store
- Platform badge strip: follows CTA, `flex flex-wrap gap-sm items-center`
- Ambient glow effect (HERO-02): deferred to Phase 5 per ROADMAP.md — Phase 4 builds structure; Phase 5 adds the glow overlay

**`PlatformBadgeStrip` visual contract:**
- Renders iOS, iPadOS, macOS platform indicator badges
- Approach: text-only labels using `TextButton` secondary variant or unstyled spans with `border border-border rounded-sm px-sm py-xs text-label text-text-muted`
- Three badges: "iPhone & iPad" + "Mac" (matching App Store badge naming) or "iOS" + "iPadOS" + "macOS"
- No SVG icons in this phase (icons are Phase 5 scope)

**`BentoCard` visual contract:**
- Container: `relative bg-[var(--bg-panel-88)] backdrop-blur-md border border-border rounded-lg p-lg shadow-card`
- Card title: 20px, weight 600, `text-text-primary`, line-height 1.2, `mb-sm`
- Card body: 16px, weight 400, `text-text-secondary`, line-height 1.55
- No hover animation in Phase 4 — hover interactions are Phase 5 polish
- Aspect ratios: two column sizes — full-width span (`col-span-2`) and half-width (`col-span-1`) — exact grid definition in `BentoGrid`

**`BentoGrid` visual contract:**
- Mobile (< 640px): single column, all cards full width, gap 16px
- Tablet (640px–1023px): 2-column grid, gap 16px, varied spans via `col-span-1` / `col-span-2`
- Desktop (≥ 1024px): 3-column grid, gap 24px, varied spans
- Grid CSS: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-md lg:gap-lg`
- Six benefit items from `messages.sections.benefits.items` — map each to a BentoCard

**`FaqAccordion` restyling contract (CONT-02):**
- Replace `border border-border bg-transparent px-4 py-3` with `border border-border bg-surface rounded-md px-lg py-md`
- Add `transition-colors duration-150` on the `<details>` background when open: `open:bg-surface-alt`
- Summary hover/active accent inversion — preserve existing pattern
- Answer text: `text-text-secondary` — preserve existing pattern

**`TextTabsNav` App Store CTA (NAV-02):**
- Add `TextButton` primary variant inside the nav's right section alongside the locale switcher
- Label: `messages.cta.appStore` ("Download on the App Store")
- Placement: right end of nav bar, before locale picker on desktop; hidden on mobile screens `< 480px` to prevent overflow (App Store CTA is in hero for mobile users)
- No changes to existing locale switching logic

## Animation Contract

Source: `FadeInSection.tsx` (Phase 2). All animation infrastructure is pre-built. Phase 4 wires section components into it.

| Element | Animation | Trigger | Variants |
|---------|-----------|---------|----------|
| Hero section | Fade in + translateY(24px → 0) | Page load (immediate, no scroll needed) | `sectionVariants`, delay 0 |
| Features section | Fade in + translateY(24px → 0) | whileInView, once | `sectionVariants` |
| Individual BentoCards | Staggered fade in + translateY(20px → 0) | whileInView, once | `staggerItemVariants`, stagger 0.1s |
| Quickstart section | Fade in + translateY(24px → 0) | whileInView, once | `sectionVariants` |
| FAQ section | Fade in + translateY(24px → 0) | whileInView, once | `sectionVariants` |
| Pricing section | Fade in + translateY(24px → 0) | whileInView, once | `sectionVariants` |
| Reduced motion | All animations skipped | `useReducedMotion()` hook in FadeInSection | `initial="visible"` bypasses all transforms |

**Properties animated:** `opacity` and `transform` (translateY) only. No layout property animation. Constraint from Phase 2 — do not violate.

**BentoCard stagger wrapper:** Needs a `'use client'` wrapper component analogous to `AppShowcaseClient` from Phase 3. Name: `BentoGridClient`. Wraps BentoGrid with `staggerContainerVariants` / `staggerItemVariants`.

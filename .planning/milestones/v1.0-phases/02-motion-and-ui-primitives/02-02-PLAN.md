1. Modify `src/app/layout.tsx`:
   - Add three new imports AFTER the existing `import type { Metadata } from 'next';` line:
     ```typescript
     import { LenisProvider } from '@/components/providers/LenisProvider';
     import { MotionProvider } from '@/components/providers/MotionProvider';
     import 'lenis/dist/lenis.css';
     ```
   - The existing `import './globals.css';` stays -- `lenis/dist/lenis.css` must be imported BEFORE `./globals.css` so globals can override if needed.
   - In the `RootLayout` function, wrap `{children}` inside body with providers:
     ```tsx
     <body>
       <MotionProvider>
         <LenisProvider>
           {children}
         </LenisProvider>
       </MotionProvider>
     </body>
     ```
   - MotionProvider is the outer wrapper (must be ancestor of all motion.* elements).
   - LenisProvider is inner (independent of motion, but nested for DOM order).
   - DO NOT add `'use client'` to layout.tsx -- it remains a server component. The providers are client components imported into the server layout (this is the standard Next.js pattern).
   - Preserve ALL existing content: metadata export, Readonly type annotation, html lang="en", etc.

2. Modify `src/app/globals.css`:
   - Find the `html` rule block (around line 73-78)
   - Remove the line `scroll-behavior: smooth;`
   - Keep `scroll-padding-top: 5.25rem;` (still needed for anchor navigation offset)
   - Keep `background: var(--color-bg);` and `color: var(--color-text-primary);`
   - Rationale: Lenis now owns scroll behavior. The CSS property is redundant when Lenis is active and could cause unexpected native smooth scroll for reduced-motion users who skip Lenis (per RESEARCH Open Question 3).
  </action>

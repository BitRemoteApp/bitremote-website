## Animation Contract

This section is Phase 2-specific. It documents the visual behavior of the three components this phase produces.

### Entrance Animation (FadeInSection)

| Property | Value | Rationale |
|----------|-------|-----------|
| Initial state | `opacity: 0, y: 24px` | 24px matches `--spacing-lg`; subtle enough to not feel exaggerated |
| Final state | `opacity: 1, y: 0` | |
| Duration | 550ms | Long enough to feel premium; short enough to not block reading |
| Easing | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` (easeOutQuart) | Decelerating curve — feels physically weighted, not mechanical |
| Trigger | `whileInView`, `once: true`, `amount: 0.08` | Fires when 8% of the element enters the viewport; fires once only |
| Stagger (child items) | 100ms between children, 50ms initial delay | Rhythm that reads as intentional, not random |
| Stagger item initial | `opacity: 0, y: 20px` | Slightly shorter travel than section-level for visual hierarchy |
| Stagger item duration | 500ms | Matches section duration closely |

**Permitted animation properties:** `opacity` and `transform` (y/translateY, scale) only. Never animate `height`, `width`, `margin`, `padding`, or any layout property. Source: VFX-05.

### Reduced Motion Behavior

| User Preference | FadeInSection | LenisProvider |
|-----------------|---------------|---------------|
| No preference (default) | Full entrance animation plays | Lenis smooth scroll active |
| `prefers-reduced-motion: reduce` | `initial` set to `'visible'` — element renders at full opacity with zero y-offset, no animation plays | `<ReactLenis>` not mounted — native browser scroll preserved |

**Source:** VFX-04, RESEARCH.md Pattern 2 + 4. Behavior is "completely absent" not "instant" — the element starts in its visible final state, so no motion ever occurs even briefly.

### Smooth Scroll (LenisProvider)

| Property | Value |
|----------|-------|
| Duration | 1.2s |
| Easing | `(t) => Math.min(1, 1.001 - Math.pow(2, -10 * t))` |
| Anchor support | `anchors: true` |
| RAF | `autoRaf: true` |
| Default state (SSR) | Lenis disabled (`skipLenis: true`) — switches to enabled on client after `matchMedia` check |

**Source:** RESEARCH.md Pattern 2, Pitfall 3 (hydration-safe default).

## Component API Contract

The three new components this phase produces. Executors must match these signatures exactly.

### LenisProvider

```
src/components/providers/LenisProvider.tsx
"use client"
export function LenisProvider({ children: ReactNode }): JSX.Element
```

- Renders `<ReactLenis root>` with options when user has no reduced-motion preference
- Renders `<>{children}</>` (passthrough) when `prefers-reduced-motion: reduce` is set
- Hydration-safe: `skipLenis` defaults to `true` on server, set on client via `useEffect`

### MotionProvider

```
src/components/providers/MotionProvider.tsx
"use client"
export function MotionProvider({ children: ReactNode }): JSX.Element
```

- Renders `<MotionConfig reducedMotion="user">{children}</MotionConfig>`
- No props beyond children

### FadeInSection

```
src/components/ui/FadeInSection.tsx
"use client"
export function FadeInSection({ children, className?, delay?, as? }): JSX.Element
```

| Prop | Type | Default | Notes |
|------|------|---------|-------|
| children | ReactNode | required | |
| className | string | undefined | Forwarded to rendered element |
| delay | number | 0 | Seconds, applied only when reduced motion is NOT active |
| as | 'section' \| 'div' \| 'article' | 'section' | Rendered element tag |

```
export const sectionVariants     — container (opacity 0→1, y 24→0)
export const staggerContainerVariants — stagger wrapper (opacity 0→1 with staggerChildren)
export const staggerItemVariants — stagger child (opacity 0→1, y 20→0)
```

Exported variants are used by client section components that need per-child stagger (Phase 4+).

### Root Layout Modification

```
src/app/layout.tsx
— server component (no "use client")
— imports: LenisProvider, MotionProvider, 'lenis/dist/lenis.css'
— wraps body children: <MotionProvider><LenisProvider>{children}</LenisProvider></MotionProvider>
```

`lenis/dist/lenis.css` must be imported here (server layout), not inside any client component.

## Hydration Safety Contract

Phase 2-specific constraint (TECH-02). All animation state must be safe for server render.

| Component | Server render behavior | Client hydration behavior |
|-----------|----------------------|--------------------------|
| MotionProvider | Renders `<MotionConfig>` wrapper — safe, no browser APIs | Same |
| LenisProvider | Renders passthrough `<>{children}</>` (skipLenis=true) | `useEffect` fires → matchMedia check → switches to `<ReactLenis>` if no reduced-motion preference |
| FadeInSection | `shouldReduceMotion` is `null` on server → treated as `false` → renders `initial="hidden"` | Hydrates matching server; reduced-motion hook fires after hydration |

No hydration mismatch will occur because: server-side defaults are conservative (no Lenis, animations enabled), and the client-side `useEffect` switches state only after React hydration is complete.

**Source:** RESEARCH.md Pitfall 1, Pitfall 3, Pattern 2.

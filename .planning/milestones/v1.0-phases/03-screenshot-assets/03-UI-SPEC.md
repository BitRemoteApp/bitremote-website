## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | `aria-hidden` button decorators, inline gaps |
| sm | 8px | Compact internal frame gaps |
| md | 16px | Default element spacing, section padding on mobile |
| lg | 24px | Frame group horizontal gaps |
| xl | 32px | Showcase section vertical padding (top/bottom of frame row) |
| 2xl | 48px | Showcase section top padding above first frame on mobile |
| 3xl | 64px | Showcase section vertical padding on desktop |

**Exceptions:**
- Device frame border: 12px (fixed structural dimension — Flowbite iPhone mockup pattern, not a spacing token)
- `scroll-padding-top` is 5.25rem (84px) — established in Phase 1 globals.css; do not change

## Color

Source: Phase 1 globals.css — full token set established. The 60/30/10 split is applied via CSS variables; exact hex values are theme-aware.

| Role | Dark value | Light value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `#0a0a0a` | `#ffffff` | Page background (`var(--color-bg)`) |
| Secondary (30%) | `#141414` | `#f8f8f8` | `var(--color-surface)` — showcase section background |
| Device frame border (surface-alt) | `#1e1e1e` | `#f0f0f0` | `var(--color-surface-alt)` — iPhone frame border and button details |
| Accent (10%) | `#3b82f6` | `#2563eb` | `var(--color-accent)` — reserved for interactive/focus elements only |
| Screen area background | `#000000` | `#000000` | Inside the frame's screen `<div>` — always black regardless of theme |

**Accent reserved for:** focus ring on interactive elements only. No accent color appears in the device frame or screenshot rendering itself.

**Note on `--bg-elevated`:** The RESEARCH.md open question is resolved here. The token does not exist. Use `var(--color-surface-alt)` for the iPhone frame border and decorative button elements. This token is `#1e1e1e` (dark) / `#f0f0f0` (light) — visually appropriate for a raised surface.

**Dark/light screenshot switching:** Implemented via HTML `<picture>` + `<source media="(prefers-color-scheme: dark)">` — zero JavaScript, zero CSS color tokens involved. This is a browser-native behavior, not a design token concern.

**`IPhoneFrame` visual contract:**
- Outer `<div>`: `relative mx-auto border-[12px] border-[var(--color-surface-alt)] bg-[var(--color-surface-alt)] rounded-[2.5rem] shadow-raise`
- Dimensions (CSS pixel): 290px wide × 628px tall (iPhone 15 proportions at ~0.74x logical size)
- Notch: `absolute top-0 left-1/2 -translate-x-1/2 w-[100px] h-[14px] bg-[var(--color-surface-alt)] rounded-b-xl aria-hidden`
- Left volume buttons (×2): `absolute -left-[14px] top-[100px] h-[40px] w-[3px] bg-[var(--color-surface-alt)] rounded-l-lg aria-hidden`
- Second left button: `top-[152px]` (same w/h)
- Right power button: `absolute -right-[14px] top-[128px] h-[60px] w-[3px] bg-[var(--color-surface-alt)] rounded-r-lg aria-hidden`
- Screen area: `rounded-[2rem] overflow-hidden w-full h-full bg-black`
- All decorative button/notch elements must carry `aria-hidden="true"`

**`AppScreenshot` visual contract:**
- Uses HTML `<picture>` element (not `next/image`)
- Source order: dark+WebP → light+WebP → `<img>` fallback (light WebP)
- `srcSet` syntax: `/screenshots/dark/slug.webp 1x, /screenshots/dark/slug@2x.webp 2x`
- `<img>`: `loading="lazy"` + `decoding="async"` + `className="w-full h-full object-cover"`
- `alt` text: descriptive, e.g. "BitRemote app home screen showing active download queue"

**`AppShowcaseClient` animation contract:**
- Wraps device frame items in `motion.div` using `staggerContainerVariants` / `staggerItemVariants` from `FadeInSection.tsx`
- Outer section wrapped in `<FadeInSection as="section" id="showcase">`
- `staggerChildren: 0.1`, `delayChildren: 0.05` (from existing variants — do not change)
- Requires `'use client'` directive

## Asset Contract

| Asset | Path | Format | Dimensions | Notes |
|-------|------|--------|------------|-------|
| iPhone screenshot — light 1x | `public/screenshots/light/iphone-home.webp` | WebP | 390px wide | Generated by `scripts/optimize-screenshots.mjs` |
| iPhone screenshot — light 2x | `public/screenshots/light/iphone-home@2x.webp` | WebP | 780px wide | Same — 2x retina |
| iPhone screenshot — dark 1x | `public/screenshots/dark/iphone-home.webp` | WebP | 390px wide | Dark OS theme variant |
| iPhone screenshot — dark 2x | `public/screenshots/dark/iphone-home@2x.webp` | WebP | 780px wide | Same — 2x retina |

**Source convention:** Raw PNG source files live in `screenshots-source/light/` and `screenshots-source/dark/` (gitignored). Only WebP output is committed to `public/`.

**Performance budget:** Each 1x WebP must be under 100KB at quality=85. The 2x variant is expected ~200-300KB. Neither should become the LCP element — the showcase section is below the hero fold, so `loading="lazy"` applies to all showcase `<img>` elements.

**WebP quality setting:** 85 (sharp option `{ quality: 85 }`). This balances file size vs. visual fidelity for app UI screenshots.

## Copywriting Contract

Phase 3 introduces minimal user-visible copy. The showcase section is primarily visual.

| Element | Copy |
|---------|------|
| Section heading (optional, discretionary) | "See it in action" |
| Screenshot alt — home screen | "BitRemote app home screen showing active downloads" |
| Screenshot alt — detail screen (if shown) | "BitRemote torrent detail view with download controls" |
| Empty state | Not applicable — screenshots are static committed assets; if a WebP file is missing, the `<img>` fallback renders the broken state at browser level |
| Error state | Not applicable — no interactive state in this component |
| Primary CTA | Not applicable — this phase adds no CTA; App Store CTA is Phase 4 |
| Destructive confirmation | None — this phase has no destructive actions |

**Accessibility copy rule:** All `aria-hidden="true"` is required on every decorative frame element (notch, buttons). The `<picture>` `<img>` fallback carries the only meaningful `alt` text per device frame.

## Animation Contract

Source: FadeInSection.tsx (Phase 2 — confirmed available).

| Element | Animation | Trigger | Variants |
|---------|-----------|---------|----------|
| Showcase section | Fade in + translateY(24px → 0) | whileInView, once | `sectionVariants` from FadeInSection |
| Individual device frames | Staggered fade in + translateY(20px → 0) | whileInView, once | `staggerItemVariants`, stagger 0.1s |
| Reduced motion | All animations skipped | `useReducedMotion()` hook | `initial="visible"` bypasses all transforms |

**Properties animated:** `opacity` and `transform` (translateY) only. No layout property animation.

|------------|--------|

## Visual Effect Specifications

### VFX-03: SvgDivider

**Implementation:** CSS `linear-gradient` on a `<div>` (not SVG ‚Äî avoids `linearGradient` ID collision on multi-divider pages).

```
height: 1px
background: linear-gradient(
  to right,
  transparent,
  var(--color-border) 25%,
  var(--color-border) 75%,
  transparent
)
aria-hidden: true
role: presentation
```

**Tailwind class form:**
```
h-px bg-[linear-gradient(to_right,transparent,var(--color-border)_25%,var(--color-border)_75%,transparent)]
```

**Usage locations:** Between every major section on the home page (5 instances). Between sections on downloader landing pages (2 instances). No `<hr>` elements remain after this phase.

**Accessibility:** `aria-hidden="true"` and `role="presentation"` ‚Äî purely decorative, not announced by screen readers.

### HERO-02: Ambient Hero Glow

**Implementation:** Two-`<div>` wrapper pattern inside `HeroSection`. The outer div clips overflow; the inner div carries the gradient.

**Outer wrapper:**
```
position: absolute
inset: 0
overflow: hidden
pointer-events: none
aria-hidden: true
```

**Inner glow div:**
```
position: absolute
top: -25% (‚àítop-1/4)
left: 50% (left-1/2)
transform: translateX(-50%)
width: 80%, max-width: 600px
aspect-ratio: 3/2
background: radial-gradient(ellipse at center, var(--hero-glow-color) 0%, transparent 70%)
filter: blur(60px)
```

**`HeroSection` section element gains:** `relative` positioning class.

**Glow appearance by theme:**
- Dark mode (`#0a0a0a` bg): `--hero-glow-color` = `color-mix(in srgb, var(--color-accent) 14%, transparent)` ‚Äî visible blue halo, not garish
- Light mode (`#ffffff` bg): `--hero-glow-color` = `color-mix(in srgb, var(--color-accent) 6%, transparent)` ‚Äî barely perceptible, prevents blue wash
- No JavaScript, no `dark:` Tailwind prefix, no hydration risk

**z-index:** Glow div is the first child of `<section id="top">`. Content siblings (wordmark, tagline, subhead, CTA) follow in DOM order and are painted on top naturally. No explicit `z-index` required.

**Mobile (375px):** At 375px viewport width, `w-[80%]` = 300px. `aspect-[3/2]` = 200px tall. Outer wrapper clips any bleed ‚Äî no horizontal scrollbar risk.

**Animation:** None. The glow is a static CSS gradient. It fades in as part of the `FadeInSection` wrapper that already wraps `HeroSection` ‚Äî no separate entrance animation needed.

## Anti-Patterns Forbidden in This Phase

| Forbidden | Reason | Use Instead |
|-----------|--------|-------------|
| `<linearGradient id="divider-grad">` in SVG | ID collision across multiple instances | CSS `linear-gradient` on `<div>` |
| `dark:opacity-[N]` Tailwind prefix on glow | Inconsistent with CSS variable token system | `--hero-glow-color` with `color-mix` in globals.css |
| `filter: blur()` without `overflow-hidden` containment | Causes horizontal scrollbar at 375px | Wrap glow in `absolute inset-0 overflow-hidden` container |
| `backdrop-filter: blur()` on glow div | Blurs content behind element, not the gradient itself | `filter: blur()` blurs the gradient spread (correct) |
| `motion/react` animation on glow | Glow is static ‚Äî animation adds `use client`, conflicts with `prefers-reduced-motion` scope | Static CSS only |
| `overflow-hidden` on `<section id="top">` outer element | Breaks any future `position: sticky` children | Apply `overflow-hidden` only to the inner glow wrapper div |
